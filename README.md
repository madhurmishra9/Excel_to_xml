# Excel ↔ XML Converter

A console tool that converts between Excel and XML **in both directions**, and
can pull remote **RSS/Atom feeds, arbitrary XML, and HTML tables** straight into
Excel or XML — from a plain URL or from cloud storage (S3 / GCS / Azure / Google
Drive).

## Features

- **Excel → XML** – one `.xml` file per sheet.
- **XML → Excel** – combines every `.xml` in a folder into one multi-sheet `.xlsx`.
- **Compare** an Excel file against a folder of XML and report differences.
- **Fetch remote sources** and convert them to XML or Excel:
  - RSS / Atom feeds (each entry → a row)
  - Arbitrary XML (the repeating record element is auto-detected and flattened)
  - HTML pages (every `<table>` → a sheet)
- **File search** across all drives (Windows).

`.xlsx`/`.xlsm` are read & written with **openpyxl**; legacy `.xls` is read with
**xlrd**. All cell values are stored as text.

## Quick start

One command sets up a local virtual environment, installs dependencies and
launches the interactive menu:

```bat
:: Windows
run.bat
```

```bash
# macOS / Linux
chmod +x run.sh && ./run.sh
```

The script is safe to re-run; it reuses the existing `.venv` on later runs.

## Manual install

```bash
pip install -r requirements.txt
# Optional, only if you fetch from cloud storage:
pip install -r requirements-cloud.txt
```

## Usage

### Interactive menu

```bash
python menu.py
```

### Command line

```bash
# Excel -> XML (one file per sheet)
python -m excel_xml excel2xml book.xlsx --out-dir out_xml

# XML folder -> single Excel workbook
python -m excel_xml xml2excel out_xml --out rebuilt.xlsx

# Compare an Excel file against a folder of XML
python -m excel_xml compare book.xlsx out_xml

# Fetch a remote source -> Excel / XML  (--kind auto|rss|xml|html|data)
python -m excel_xml fetch2excel https://hnrss.org/frontpage --out hn.xlsx
python -m excel_xml fetch2xml  https://www.w3schools.com/xml/note.xml --out-dir note_xml
python -m excel_xml fetch2excel https://example.com/page.html --kind html --out tables.xlsx

# Search for a file (Windows)
python -m excel_xml search report.xlsx
```

### Cloud sources

`fetch2excel` / `fetch2xml` accept cloud URIs as the source. Credentials come
from each provider's standard configuration:

| Scheme                                   | Backend                | Credentials |
|------------------------------------------|------------------------|-------------|
| `s3://bucket/key`                        | boto3                  | AWS env vars / `~/.aws` |
| `gs://bucket/key`                        | google-cloud-storage   | `GOOGLE_APPLICATION_CREDENTIALS` |
| `gdrive://<id>` or a Drive share URL     | gdown                  | public files |
| `az://container/blob` or a blob URL      | azure-storage-blob     | `AZURE_STORAGE_CONNECTION_STRING` |

## XML format

```xml
<data>
  <Sheet1 name="Sheet 1">
    <row1>
      <column1>value</column1>
      <column2>value</column2>
    </row1>
  </Sheet1>
</data>
```

The real sheet name is stored in the `name` attribute so names with spaces or
special characters round-trip correctly; files generated by the original tool
(tag-as-name, no attribute) are still read.
